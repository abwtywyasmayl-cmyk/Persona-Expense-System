# نظام تتبع المصروفات الشخصي (Simple Expenses Tracker)

نظام ويب بسيط مبني بلغة **PHP** وقاعدة بيانات **MySQL** لتتبع الدخل والمصروفات الشخصية، مع واجهة عربية بالكامل تدعم اتجاه الكتابة من اليمين لليسار (RTL).

---

## 📋 نظرة عامة

يتيح النظام للمستخدم (بعد تسجيل الدخول) القيام بما يلي:
- عرض لوحة تحكم تلخص الرصيد الكلي، إجمالي الدخل، إجمالي المصروفات، وعدد الفئات.
- إضافة معاملات مالية جديدة (دخل أو مصروف).
- عرض جميع المعاملات في جدول مع إمكانية التعديل والحذف.
- تسجيل الخروج وإنهاء الجلسة بأمان.

---

## 🗂️ هيكلة المشروع

```
logout/

├── index.php              # لوحة التحكم الرئيسية (Dashboard)

├── edit_transaction.php   # عرض/تعديل معاملة موجودة
└── expenses_db.sql        # سكربت إنشاء قاعدة البيانات والجدول والبيانات التجريبية
```

## 🚀 خطوات التشغيل (محلياً باستخدام XAMPP)

1. انسخ مجلد المشروع بالكامل إلى `htdocs` (في حال استخدام XAMPP).
2. شغّل **Apache** و **MySQL** من لوحة تحكم XAMPP.
3. افتح **phpMyAdmin** ونفّذ ملف `expenses_db.sql` لإنشاء قاعدة البيانات `simple_expenses_db` والجدول `transactions` مع بيانات تجريبية.
4. تأكد من إعدادات الاتصال بقاعدة البيانات في `config.php`:
   ```php
   define('DB_SERVER', 'localhost');
   define('DB_USERNAME', 'root');
   define('DB_PASSWORD', '');
   define('DB_NAME', 'simple_expenses_db');
   ```
5. افتح المتصفح على: `http://localhost/logout/login.php`

## 🗄️ هيكل قاعدة البيانات

**قاعدة البيانات:** `simple_expenses_db`
**الجدول:** `transactions`

| العمود | النوع | الوصف |
|---|---|---|
| id | INT (PK, AUTO_INCREMENT) | المعرّف الفريد للمعاملة |
| description | VARCHAR(255) | وصف المعاملة |
| amount | DECIMAL(10,2) | المبلغ (موجب = دخل، سالب = مصروف) |
| category | ENUM | food, transport, entertainment, income, bills, shopping, health, other |
| date | DATE | تاريخ المعاملة |
| payment_method | ENUM | cash, credit card, debit card, digital wallet |
| created_at | TIMESTAMP | تاريخ إنشاء السجل تلقائياً |
| updated_at | TIMESTAMP | تاريخ آخر تعديل تلقائياً |

## ⚠️ ملاحظات أمنية هامة

هذا المشروع **تعليمي/تجريبي** ويحتوي على ثغرات أمنية يجب معالجتها قبل استخدامه في بيئة إنتاج حقيقية:

1. **حقن SQL (SQL Injection):** الاستعلامات في `add_transaction.php`، `edit_transaction.php`، و `transactions.php` تُبنى بدمج مباشر لمدخلات المستخدم (`$_POST` / `$_GET`) دون استخدام Prepared Statements، مما يسمح بحقن أوامر SQL ضارة.
2. **عدم تعقيم المخرجات (XSS):** بيانات مثل `description` و `username` تُطبع مباشرة في HTML دون `htmlspecialchars()`، مما يفتح الباب أمام هجمات Cross-Site Scripting.
3. **بيانات دخول ثابتة:** اسم المستخدم وكلمة المرور مكتوبة مباشرة في الكود (Hard-coded) وبدون تشفير (Hashing).
4. **لا يوجد حماية CSRF:** نماذج الإضافة والتعديل لا تحتوي على رمز تحقق (CSRF Token).

**التوصية:** استخدام `mysqli` مع Prepared Statements أو الانتقال إلى PDO، وتطبيق `password_hash()` / `password_verify()` لإدارة كلمات المرور.
